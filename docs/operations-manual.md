# 会場運用マニュアル (当日スタッフ向け)

## 準備 (前日まで)

- [ ] 各PCのブラウザで `https://YukiInoueNakata.github.io/idol-diagnostic-test/` が開けるか確認
- [ ] 各PCでテスト受検を1回実施して印刷が出るか確認
- [ ] 各PCの既定プリンターが当日使うプリンターになっているか確認 (Windows 設定 → プリンターとスキャナー → 既定値)
- [ ] 会場Wi-Fiで各PCがインターネット接続できるか確認
- [ ] スプレッドシートに行が追加されることを確認

## 会場セットアップ

1. 各PCで Chrome / Edge を起動
2. トップページを開く
3. ブラウザをキオスクモード (F11) にして受検者以外の操作を防ぐ (任意)
4. プリンターに用紙をセット，電源ON

## 受検1名あたりの流れ (スタッフ補助)

1. 受検者が席に着く
2. スタッフが index.html を表示状態にする (前回の結果が残っていたら ×ボタンで終了 → トップへ)
3. 受検者が年齢・性別・グループ・質問セットを選んで「診断を開始」
4. 10問に回答
5. 結果画面で「印刷して終了」
6. 印刷物を受検者に渡す

## トラブル対応

### ブラウザが固まった
- タブを閉じて再度開く (結果は既にスプレッドシートに送信済みなら失われない)
- ネットワーク接続を確認

### 印刷が出ない
- プリンター電源・用紙・インクを確認
- 結果画面で再度「印刷して終了」
- ダメなら，結果画面のURLをメモしてあとで再印刷可能

### ネットが切れた
- 診断自体はオフラインで継続可能 (採点ロジックはブラウザ内)
- 結果は端末の localStorage に一時保存され，ネット復活後に自動再送
- 印刷もオフラインで可能

### スプレッドシートに記録されていない
- 各PCで F12 (開発者ツール) → Console → `localStorage.getItem("idol_pending_submissions")` を確認
- 中身があればオフラインキューに溜まっている．ネットが戻れば自動で送信される
- 強制送信: ページをリロード

## 当日の全体確認用

- スプレッドシートの行数を時間ごとに確認 (例: 1時間後 / 3時間後 / 終了時) ，期待値と合うか
- 異常な値 (big5 が全部同じ，matched が NULL 等) がないか目視確認
